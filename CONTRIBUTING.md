# Contributing to SmartDrobe 🎉

Thank you for your interest in contributing to SmartDrobe! We welcome contributions from the community.

## 🌟 Ways to Contribute

- 🐛 **Report Bugs** - Found a bug? [Open an issue](https://github.com/anushkatiwari0/Smartdrobe/issues/new)
- ✨ **Suggest Features** - Have an idea? Share it in [Issues](https://github.com/anushkatiwari0/Smartdrobe/issues)
- 📝 **Improve Documentation** - Help make our docs better
- 💻 **Submit Code** - Fix bugs or add features via pull requests
- 🎨 **Design Improvements** - UI/UX enhancements are welcome

## 🚀 Getting Started

### 1. Fork & Clone

```bash
# Fork the repository on GitHub, then:
git clone https://github.com/YOUR_USERNAME/Smartdrobe.git
cd Smartdrobe
```

### 2. Set Up Development Environment

```bash
# Install dependencies
npm install

# Copy environment variables
cp .env.example .env.local
# Fill in your API keys in .env.local

# Start development server
npm run dev
```

### 3. Create a Branch

```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/bug-description
```

## 📋 Development Guidelines

### Code Style

- **TypeScript** - Use TypeScript for type safety
- **ESLint** - Run `npm run lint` before committing
- **Formatting** - Code should be clean and readable
- **Comments** - Add comments for complex logic

### Commit Messages

Follow conventional commits format:

```
feat: add dark mode toggle
fix: resolve login authentication issue
docs: update README with new features
style: format code with prettier
refactor: simplify outfit generation logic
test: add tests for wardrobe API
chore: update dependencies
```

### Pull Request Process

1. **Update Documentation** - If you change functionality, update docs
2. **Test Your Changes** - Ensure everything works
3. **Run Linting** - `npm run lint` and `npm run typecheck`
4. **Create PR** - Write a clear description of your changes
5. **Link Issues** - Reference any related issues (e.g., "Fixes #123")

## 🧪 Testing

```bash
# Run type checking
npm run typecheck

# Run linter
npm run lint

# Test the build
npm run build
```

## 📝 Documentation

When adding features:

- Update README.md if needed
- Add JSDoc comments to functions
- Update PRD.md for major features
- Include usage examples

## 🤝 Code of Conduct

### Our Standards

- ✅ Be respectful and inclusive
- ✅ Welcome newcomers
- ✅ Provide constructive feedback
- ✅ Focus on what's best for the community
- ❌ No harassment or trolling
- ❌ No spam or self-promotion

### Reporting Issues

If you experience unacceptable behavior, please report it by emailing the project maintainer or opening a private issue.

## 🐛 Bug Reports

When reporting bugs, include:

- **Description** - What happened vs. what you expected
- **Steps to Reproduce** - Detailed steps to recreate the issue
- **Environment** - OS, Node version, browser
- **Screenshots** - If applicable
- **Error Messages** - Full error logs

## ✨ Feature Requests

When suggesting features:

- **Use Case** - Why is this feature needed?
- **Proposed Solution** - How should it work?
- **Alternatives** - Other approaches you've considered
- **Mockups** - Wireframes or designs if applicable

## 🎨 UI/UX Contributions

- Follow the existing design system (Tailwind CSS + shadcn/ui)
- Maintain accessibility standards
- Ensure responsive design (mobile, tablet, desktop)
- Test dark mode compatibility

## 🔒 Security

**DO NOT** open public issues for security vulnerabilities.

Instead:
- Email the maintainer privately
- Include detailed description and reproduction steps
- Allow time for a fix before public disclosure

See [SECURITY.md](SECURITY.md) for details.
